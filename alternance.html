Alternance · 2025 – 2026 AP-HP Henri Mondor

Plateforme ATLAS — GenoBioMICS

Mon alternance porte sur la plateforme ATLAS, une solution technologique hospitalière dédiée au séquençage ADN, à l'organisation des analyses bioinformatiques et à l'exploitation des données pour l'IA. Mon rôle est de construire le middleware central qui relie les applications, les identités utilisateurs, les autorisations, les données de laboratoire et les outils d'analyse, afin de fournir une plateforme fiable, sécurisée et exploitable par les équipes médicales, techniques et de recherche.

Missions & Réalisations

🔐 IAM & Sécurité

  • Authentification OIDC avec Zitadel (PKCE, MFA, JWT, organisations)
  • Sessions côté serveur HTTPOnly — aucun token dans le navigateur
  • Autorisations ReBAC via SpiceDB (tuples de relations)
  • Autorisations ABAC via OPA (règles Rego contextuelles)
  • 6 rôles métier : admin, rh, gestionnaire, technicien, bioinformaticien, prescripteur

⚙️ Backend Go (BFF)

  • Pattern BFF : le frontend ne contacte jamais directement les services tiers
  • API REST complète — endpoints protégés et publics
  • Pipeline pré-inscription : formulaire → validation admin → activation compte
  • Intégrations : Dgraph, SpiceDB, OPA, PostgreSQL, SeaweedFS, OpenSearch, Redis

🎨 Frontend React/TypeScript

  • SPA React 19 avec authentification par cookie de session HTTPOnly
  • Modules : portail, administration, pré-inscription RH, pipeline kanban
  • Panel admin : observabilité live, gestion utilisateurs, matrice permissions
  • Hub analyses : demandes d'analyse NGS, catalogue, devis, pipeline kanban

🗃️ DataLake & DataHouse

  • Stockage objet S3 via SeaweedFS (FASTQ bruts, pièces jointes, exports)
  • Pipeline ETL Bronze → Silver → Gold avec DuckDB + Python
  • Tables analytiques versionnées Apache Iceberg via catalogue Nessie
  • Moteur SQL analytique Trino pour requêtes sur couche Gold
  • Intégration pipelines HPC NGS : séquenceur → SeaweedFS → ETL → Iceberg

🔭 Observabilité intégrée

  • Collecte métriques Prometheus + persistance Redis time-series 7 jours
  • Historique alertes CPU/MEM/NET et uptime health (Redis)
  • Sessions utilisateurs actives en temps réel (heartbeat Redis)
  • Panel admin dédié avec stats par route, latence et erreurs

🗄️ Données graphe (Dgraph)

  • Modélisation des entités hospitalières : Personne, Poste, Unité, Contrat, Diplôme…
  • CMDB dédié : inventaire infra (services, clusters, pipelines IA, politiques)
  • Schéma GraphQL + DQL, seeds, outils Ratel UI / Voyager / Altair
  • Indexation full-text et audit via OpenSearch
